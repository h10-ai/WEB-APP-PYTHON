# WEB-APP-PYTHON


*PURPOSE* :- A PYTHON BASED UI/UX WEB APP DESIGN
             FOR IMPLEMENTATION OF A VOICE ENABLED CHATBOT .

*GOAL* :- 1-IMPLEMENT GEMINI API KEY TO GENERATE AND TALK IN REAL TIME 
          2- TECHY INSPIRED UI(USER INTERFACE) AND SEAMLESS UX(USER EXPERIENCE)      
          3-CHAT HISTORY UPTO 5 CHATS 
          4- DIFFERENT VOICES ENABLED 
          5- SUCESSFULLY DEPLOY IN WEB 

 *TARGET USER* :- STUDENT AND FACULTIES (i.e easy ,clean and seamless use)

 *FUNCTIONAL REQUIREMENTS* :- 1.CHATBOT HEARS QUERIES --> GRADIENT ANIMATIONS ---> REPLIES BACK IN THE SAME WAY
                              2.ALWAYS WAITS 2-4 SECOND BEFORE REPLYING . (i.e reply latency should be maintained purposely) 
                              3.SAVES CHAT HISTORY UPTO 5 CHATS . 
                              4. SEAMLESS CHANGE BETWEEN "TEXT BASED CHAT" AND "SPEECH TO SPEECH" (i.e there will two interfaces interchanging by user preference . *look sidebar ui in most AIs)
                              5.***SPEECH SHOULD RESULT INTO GENERATED TEXT NO MATTER WHICH INTERFACE USER SELECTS***



 *NON-FUNCTIONAL REQUIREMENTS* :- 1.OPENING ANIMATIONS WHEN OPENING THE WEB APP (look for motion graphics to be used during development)
                                  2.GRADIENT ANIMATION WHILE SPEECH BASED CHAT (LOOK CHATGPT VOICE GRADIENT VISUALIZER )
